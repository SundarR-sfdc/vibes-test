Account Flex Card — Design & Admin Guide

Overview

Purpose: A compact Lightning Web Component (LWC) used on Account record pages to display key account details in a flex-card style. It is admin-configurable from the Lightning App Builder and shows optional related lists (Contacts, Opportunities), an icon/logo, and quick actions (Edit/Open/View All related lists).

Location (code): force-app/main/default/lwc/accountFlexCard/

Deployed to: org alias vsCodeOrg.

  • Show Name
  • Show Industry
  • Show Phone
  • Show Website
  • Show Billing Address
  • Show Annual Revenue
  • Show Contacts (related list)
  • Show Opportunities (related list)
  • Logo URL (optional)

How to change settings (quick steps)

  1. Open an Account record → Gear → Edit Page (Lightning App Builder).
  2. Select accountFlexCard on the canvas.
  3. Use the Properties pane on the right to toggle fields and paste Logo URL.
  4. Set Component Visibility if needed.
  5. Save → Back → Refresh Account record.

Design details — @api usage and rationale

The component declares public properties with @api. These are mapped to metadata <property> entries in the component's .js-meta.xml file, and appear in App Builder as editable controls. This makes the component declarative and admin-configurable without redeploy.

Comparison with alternatives

Custom Metadata / Settings: Good for global defaults but not per-page customization. Requires extra setup and often Apex to fetch.

Developer-only config: Simple but requires redeploy for every change; not admin-friendly.

Troubleshooting

  • Component not listed: ensure .js-meta.xml has <isExposed>true</isExposed> and targets.
  • Blank fields: check Field-Level Security for the running user.
  • Logo blocked: upload as static resource or ensure CSP allows external images.
